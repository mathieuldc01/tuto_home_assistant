    Préparation de la Raspberry Pi

    Installation de Debian sur le Raspberry Pi

    Télécharger le logiciel officiel :

    Raspberry Pi Imager

    Une fois le logiciel installé :

    1. Sélectionner le modèle de Raspberry Pi
    2. Choisir le système d'exploitation (Debian x64)
    3. Sélectionner la carte micro SD
    4. Configurer les paramètres (utilisateur, mot de passe, réseau)
    5. Lancer l'installation

    Une fois l'installation terminée, insérer la carte micro-SD dans le Raspberry Pi puis démarrer l'appareil.

    Connexion au Raspberry Pi

    Depuis un ordinateur connecté au même réseau :

    Ouvrir une invite de commande :

    Win + R → cmd → Entrée

    Connexion SSH :

    ssh NOMUTILISATEUR@NOMRASPBERRY

    Entrer ensuite le mot de passe du Raspberry Pi.

    Installation du conteneur Home Assistant

    Télécharger le dépôt contenant le conteneur :

    git clone https://gitlab.ec-lyon.fr/durandu/temperatures-monitoring.git

    Entrer dans le dossier :

    cd temperatures-monitoring

    Construire le conteneur Docker :

    docker build -t NOM_DU_CONTENEUR

    Lancer le conteneur :

    sudo docker compose up -d
    Attention : vous devez être dans le dossier temperatures-monitoring pour lancer le conteneur.
    Connexion à Home Assistant

    Récupérer l'adresse IP du Raspberry Pi :

    hostname -I

    Dans un navigateur web, entrer :

    ADRESSE_IP:8123

    Vous accéderez alors à l'interface de Home Assistant.

    Connexion du Raspberry Pi à un nouveau réseau WiFi

    Lister les réseaux disponibles :

    nmcli device wifi list

    Se connecter à un réseau :

    sudo nmcli device wifi connect "NOM_DU_RESEAU" password "MOT_DE_PASSE"

    Ajouter des Capteur

    En gros plusieurs méthode possible, soit on relie directement nos capeur en filaire sur le rasberry Pi soit on peut les relié en sans-fil (utilisation d'un esp32)
    Connexion d'un capteur 1-Wire

    Branchement du capteur 1-Wire :

    Connexion au Raspberry Pi / ESP32 :

    • VCC → borne VCC
    • GND → borne GND
    • DQ (Data) → un des GPIO
    Activation et vérification du protocole 1-Wire Sur Rasberry Pi

    Vérifier que le module est actif :

    lsmod | grep w1

    Si rien ne s'affiche, modifier le fichier de configuration :

    sudo nano /boot/config.txt

    Ajouter la ligne :

    dtoverlay=w1-gpio

    Redémarrer le Raspberry Pi :

    sudo reboot

    Lister les capteurs détectés :

    cd /sys/bus/w1/devices/ ls
    Installation et configuration de OWServer

    Installer OWServer :

    sudo apt-get update sudo apt-get install owserver

    Modifier le fichier de configuration :

    sudo nano /etc/owfs.conf

    Ajouter la ligne suivante à la fin :

    address = 0.0.0.0

    Redémarrer le service :

    sudo systemctl restart owserver
    Ajout d'un capteur 1Wire sur Rasberry Pi dans Home Assistant

    Dans Home Assistant :

    • Paramètres
    • Appareils et services
    • 1Wire

    Cliquer sur Ajouter un pont puis entrer :

    • Adresse : IP du Raspberry Pi
    • Port : 4304
    Ajouter un ESP32 à Home Assistant

    Il faudra au préalble flasher votre ESP32 pour pouvoir l’ajouter à Home Assistant.(Example de Flash dans la partie suivante)

    Étape 1 : Préparer Home Assistant

    Assurez-vous que Home Assistant est installé et en fonctionnement. Dans l’interface web :

    • Allez dans Paramètres
    • Ensuite Appareils et Services
    • Cliquez sur Ajouter une intégration

    Étape 2 : Ajouter ESPHome

    Recherchez et sélectionnez ESPHome comme type d’intégration.

    Entrez l’adresse IP ou le nom du ESP32 que vous avez flashé et validez. Si votre ESP32 et Home Assistant sont sur le même réseau, il devrait apparaître automatiquement.

    Étape 3 : Vérification

    Dans Home Assistant, ouvrez le tableau de bord et vérifiez que les entités liées à votre ESP32 apparaissent correctement.

    Si les capteurs ne s’affichent pas, assurez-vous que :

    • L’ESP32 est connecté au réseau WiFi
    • Le firmware ESPHome contient l’API activée :
    api:

    Créer et Ajouter une Map

    Maintenant que nos capteur son paramétré, il faut pouvoir afficher leur mesure, pour cela, on utilise une visualisation sur map. On va voir comment créer et ajouter des map a notre homeAssistance.
    Création d'une map
    Ajout d'un capteur sur la carte

    Ajouter le code suivant dans la section rules :

    - name: Temperature Salle XXXX entities: - sensor.NOMDUCAPTEUR element: NOMDEL'ELEMENT state_action: - service: floorplan.text_set service_data: >- ${(entity.state !== undefined) ? Math.round(entity.state * 10) / 10 + "°C" : "??°C"} - service: floorplan.class_set service_data: class: static-temp

    Ajouter également le capteur dans la liste des entities.

    Ajouter une nouvelle carte sur le Raspberry Pi

    Envoyer un fichier sur le Raspberry Pi :

    scp "CHEMIN_DU_FICHIER" lesframboises@IP_DU_RASPBERRY:/home/lesframboises/

    Se connecter ensuite au Raspberry Pi :

    ssh lesframboises@LesFramboises

    Déplacer la carte dans le bon dossier :

    sudo mv ~/NOMDUFICHIER /home/lesframboises/temperatures-monitoring/config/www/floorplan/
